Why Use Packages
Encapsulation: Keep related logic together (e.g., all claim procedures in one package).

Reusability: Call functions/procedures from anywhere without rewriting.

Security: Expose only the specification (interface), hide implementation details in the body.

Performance: Once loaded, package code stays in memory, reducing overhead.
